Space X rocket landing predictor

May 4, 2022

The Space X company offers their service of special missions and have rockets with a cost of 62 million dollars, they save a lot of money because the rocket’s first stage is reusable if landing successfully, since the most popular model is Falcon 9, we’re going to focus on this rocket model. The Space X webpage contain many features of all the space mission made up until today that can help us in order to predict if the first stage will land successfully and save a lot of money to the company.

Data collection

  • The data was collected from Wikipedia using BeautifulSoup to web scrape the site, using the requests we got an html text which was filtered in order to get the relevant information from it such as booster version, landing outcome, payload mass, etc. Then we parsed it to a pandas data frame.
  • We also made use of Space X API, we made a REST call getting a json file as response which decoded with json function and converted to a pandas data frame using json normalization function.

Data cleaning

  • The extracted data was stripped in order to eliminate unnecessary and noisy characters, next we searched missing values and replaced it with either 0 for booster landing or average value for the payload mass, furthermore we formatted the variables to its correct format, finally we made a one-hot encoding for categorical variables.

EDA and Data visualization

  • Using a scatter plot between launch site vs number of flight we can observe that CCAFSSLC 40 have more successful landings when the fight number is higher. Whereas the other two sites doesn’t seem to have the same correlation.
  • Doing the same with payload mass, we can see that heavy payloads aren’t launched at VAFB-SLC launch site
  • Looking at the relationship between success rate and orbit type, We can see that ES-L1, GEO, HEO and SSO have a 100% success rate, a huge gap with VLEO, the second highest, which only perform 80%.
  • Since orbit seems to be an important variable, let’s see it correlation with other variables.

LEO orbit the Success appears related to the number of flights; on the other hand, there seems to be no relationship between flight number when in GTO orbit.

  • With heavy payloads the successful landing or positive landing rate are more for Polar,LEO and ISS. However for GTO we cannot distinguish this well as both positive landing rate and negative landing(unsuccessful mission) are both there here.
  • If we observe the tendency through time, is evident that the success landings have been increasing since 2013.
  • Plotting the launch sites on a map we can easily observe that all of them are near the coast and there’s just one launch site in the west coast whereas the east coast have 3.
  • Adding a cluster marker we see that east coast have +four times more launches than west coast, CCAFS SLC-40 site is the one with least launches and CCAFS LC-40 is the one with most even when they’re next to each other. But if we visually look at the success rate, we can see KSC LC-39A site has the highest and CCAFS LC-40 site has the lowest.

To summarize all the relevant insights we got, we created an interactive dashboard with python and dash which displays, when all launch sites are selected, the proportion of the total launches by site on a pie chart and the correlation between payload and outcome with a scatter plot which can be changed depending on the range of payload mass; when a specific launch site is selected, the pie chart shows the proportion of successful and failed outcomes as well the relationship between payload mass and outcomes.

Building the classification model

  • To build the model we used Sklearn and Numpy libraries, and Matplotlib to measure accuracy.

First we standardize the data. Next, we split the data into training and testing data using the function train_test_split. The training data is divided into validation data, a second set used for training data; then the models are trained and hyperparameters are selected using the function GridSearchCV.

  • The first model we build was a Logistic regression with penalty of 12 which achieved a training accuracy of 84.64% and a test accuracy of 83.33%. Then we plotted the confusion matrix.
  • The second model we build was a Support vector machine (SVM) with sigmoid kernel which achieved a training accuracy of 84.82% and a test accuracy of 83.33%. Then we plotted the confusion matrix.
  • The third model we build was a Decision tree with depth of 8 which achieved a training accuracy of 87.68% and a test accuracy of 83.33%. Then we plotted the confusion matrix.
  • The last model we build was a K nearest neighbors with 10 neighbors which achieved a training accuracy of 84.82% and a test accuracy of 83.33%. Then we plotted the confusion matrix.

Since test accuracy was the same for all models, the measure we used to get the best model was training accuracy. The best model is Decision Tree with training accuracy of 87.68% and test accuracy of 83.33%.
